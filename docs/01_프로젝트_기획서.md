# 📋 프로젝트 기획서 - Project Tracker

## 문서 정보
- **프로젝트명**: Project Tracker (프로젝트 진행률 관리 도구)
- **개발자**: 구현모
- **개발 기간**: 7일
- **개발 목적**: 학교 수업 프로젝트 + 실제 사용
- **버전**: 1.0
- **작성일**: 2025년 11월 23일

---

## 1. 프로젝트 개요

### 1.1 배경 및 문제 인식

**현재 상황**
- iOS 부트캠프 진행 중
- 여러 프로젝트 동시 진행 (감정 일기 앱 등)
- 기존 관리 도구의 문제점:
  - 너무 복잡하거나 (Jira, Notion 프로젝트)
  - 너무 단순하거나 (메모앱, 스프레드시트)
  - 부트캠프 특성에 맞지 않음

**불편했던 점**
1. 프로젝트별로 태스크가 흩어져 있음
2. 전체 진행률을 한눈에 보기 어려움
3. 완료한 것과 남은 것 구분이 불명확
4. 학습 내용과 프로젝트 연결이 안 됨
5. 회고를 제대로 기록하지 못함

**필요성**
- 부트캠프 학생에게 최적화된 경량 도구
- 빠르게 태스크 추가/이동 가능
- 시각적으로 진행 상황 파악
- 간단한 회고 기록

---

### 1.2 프로젝트 목적

**핵심 목표**
> "부트캠프 학생이 프로젝트를 체계적으로 관리하고, 진행 상황을 시각적으로 파악할 수 있는 심플한 도구 제공"

**구체적 목표**
1. **태스크 관리**: Kanban 보드로 직관적 관리
2. **진행률 시각화**: 차트로 한눈에 파악
3. **회고 기록**: 프로젝트 완료 후 학습 내용 정리
4. **간편함**: 클릭 몇 번으로 모든 작업 완료

---

### 1.3 타겟 사용자

**주 사용자**: 부트캠프/코딩 교육 수강생
- 여러 프로젝트 동시 진행
- 빠른 개발 사이클
- 학습과 프로젝트의 병행

**부 사용자**: 
- 사이드 프로젝트 진행하는 개발자
- 팀 프로젝트 관리자

---

## 4. 기능 상세 명세

### 4.1 프로젝트 관리

#### 프로젝트 생성
**입력 필드**:
- 프로젝트명 (필수)
- 설명 (선택)
- 시작일 (기본: 오늘)
- 목표 완료일 (선택)
- 상태 (진행중/완료/보류)

**자동 생성**:
- 생성일시
- 마지막 업데이트 일시

#### 프로젝트 전환
- 사이드바 또는 드롭다운에서 선택
- 현재 프로젝트 하이라이트
- 최근 작업한 프로젝트 3개 빠른 접근

---

### 4.2 Kanban 보드

#### 컬럼 구조
```
1. To Do (할 일)
   - 아직 시작 안 한 태스크
   - 우선순위 표시 가능

2. In Progress (진행 중)
   - 현재 작업 중인 태스크
   - 시작 시간 자동 기록

3. Done (완료)
   - 완료된 태스크
   - 완료 시간 자동 기록
```

#### 태스크 카드 구성
```
┌─────────────────────────┐
│ ☐ 태스크 제목           │
│ 💻 Dev | 🔴 High        │
│ 📅 2024-11-25           │
│ [상세] [수정] [삭제]    │
└─────────────────────────┘
```

**표시 정보**:
- 제목
- 태그 (아이콘)
- 우선순위 (색상)
- 마감일
- 액션 버튼

#### 상태 변경
- **드래그 앤 드롭**: 컬럼 간 이동
- **버튼 클릭**: 다음/이전 단계로 이동
- **자동 기록**: 상태 변경 시간 저장

---

### 4.4 진행률 대시보드

#### 메트릭 카드
```
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│ 전체    │ │ 완료    │ │ 진행률  │ │ 남은 일 │
│ 15      │ │ 8       │ │ 53%     │ │ 3일     │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
```

#### 차트
1. **진행률 추이 (선 그래프)**
   - X축: 날짜
   - Y축: 완료율 (%)
   - 목표선 표시

2. **상태별 분포 (원형 차트)**
   - To Do
   - In Progress  
   - Done

3. **태그별 분포 (막대 차트)**
   - 어떤 작업을 많이 했는지

4. **일별 완료 태스크 (막대 차트)**
   - 작업 패턴 파악

### 4.5 마일스톤 관리

**마일스톤이란?**
- 프로젝트의 주요 체크포인트
- 예: MVP 완성, 베타 테스트, 최종 발표

**기능**:
- 마일스톤 생성/수정/삭제
- 날짜 및 설명 설정
- 완료 여부 체크
- 대시보드에 표시

**예시**:
```
☑ 기획 완료 (2024-11-10) ✅ 달성
☑ MVP 개발 (2024-11-20) ✅ 달성  
☐ 베타 테스트 (2024-11-27) ⏳ 진행 중
☐ 최종 발표 (2024-12-01) 📅 예정
```

### 4.6 회고 기록 (KPT)

**KPT란?**
- **Keep**: 잘했던 것, 계속할 것
- **Problem**: 문제점, 개선 필요
- **Try**: 다음에 시도할 것

**입력 양식**:
```
Keep (계속할 것)
┌─────────────────────────┐
│ - 매일 스탠드업 미팅    │
│ - 커밋 메시지 규칙 준수 │
└─────────────────────────┘

Problem (문제점)
┌─────────────────────────┐
│ - 일정 예측 부정확      │
│ - 테스트 코드 부족      │
└─────────────────────────┘

Try (시도할 것)
┌─────────────────────────┐
│ - 버퍼 시간 20% 추가    │
│ - TDD 방식 시도         │
└─────────────────────────┘

학습 내용
┌─────────────────────────┐
│ - MVVM 패턴 적용        │
│ - Combine 프레임워크    │
└─────────────────────────┘
```

**저장 및 조회**:
- 프로젝트별 회고 저장
- 과거 회고 조회
- Markdown 내보내기
